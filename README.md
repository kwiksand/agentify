# agentify

A small toolkit for bootstrapping a project with portable, multi-platform
AI coding-assistant configuration. One canonical set of sub-agent
definitions, transpiled into the native formats used by the major
agentic CLIs and IDE assistants.

Run `agentify.sh` in any project directory and you get:

- An `AGENTS.md` and `README.md` written from your project's detected
  stack.
- A canonical `agents/` directory of sub-agent definitions.
- Starter Claude Code skills under `.claude/skills/`.
- Per-platform agent files generated by `sync/sync.py` for every
  supported tool (see below).

## Why

Every coding assistant has invented its own sub-agent / chat-mode /
recipe / steering format. The *content* is ~90% the same — a name, a
description, a system prompt, a tool allowlist, sometimes a model hint.
This repo keeps **one source of truth** (Claude Code's frontmatter
schema, since it's the strictest superset) and adapters fan it out to
each platform's native layout.

## Supported AI tools

| Tool | Output location | Format |
|------|-----------------|--------|
| **Claude Code** (Anthropic CLI / IDE) | `.claude/agents/<name>.md` | Native sub-agent markdown + YAML frontmatter |
| **GitHub Copilot** (custom agents) | `.github/agents/<name>.agent.md` | Copilot custom-agent markdown |
| **Kiro** (AWS) | `.kiro/steering/<name>.md` | Steering doc with `inclusion: manual` |
| **Goose** (Block) | `.goose/recipes/<name>.yaml` | Goose recipe YAML |

Tools that read `AGENTS.md` directly (Cursor, Aider, OpenAI Codex CLI,
and others that follow the [agents.md](https://agents.md) convention)
are covered by the top-level `AGENTS.md` that `agentify.sh` writes.

Shared MCP server config lives in `mcp.json` at the repo root and can
be symlinked / pointed at by each platform — see [Tools via MCP](#tools-via-mcp).

## Repo layout

```
agentify.sh                   # scaffolder: detect stack, render templates, run sync
templates/
  AGENTS.md.tpl               # rendered into the target project as AGENTS.md
  README.md.tpl               # rendered into the target project as README.md
  SKILL.md.tpl                # rendered into .claude/skills/<name>/SKILL.md

agents/                       # canonical sub-agent source of truth
  code-writer.md
  debugger.md
  environment-setup.md
  tester.md
  version-control.md

sync/
  sync.py                     # entry point: reads agents/, writes target dirs
  adapters/
    claude_code.py
    copilot.py
    goose.py
    kiro.py

examples/                     # rendered output, committed for reference
  .claude/agents/
  .github/agents/
  .goose/recipes/
  .kiro/steering/

AGENTS.md                     # repo-level instructions (this repo, for AI tools)
mcp.json                      # shared MCP server config
```

## Usage

### Scaffold a project

From inside the project you want to set up:

```bash
/path/to/agentify.sh
```

Or point it at a directory:

```bash
agentify.sh --dir ~/code/my-service
agentify.sh --dir ~/code/my-service --name my-service --yes   # accept all defaults
agentify.sh --no-sync                                         # scaffold without running sync.py
```

`agentify.sh` detects the language, frameworks, and build/test command
from common manifest files (`package.json`, `pyproject.toml`, `go.mod`,
`Cargo.toml`, etc.), prompts to fill in the gaps, and renders the
templates. It then invokes `sync/sync.py` to fan agent definitions out
to every platform.

### Re-sync agents only

When you edit an `agents/*.md` file or add a new one, push the change
to every platform:

```bash
python3 sync/sync.py                              # write everything
python3 sync/sync.py --dry-run                    # preview only
python3 sync/sync.py --target claude-code         # one platform
python3 sync/sync.py --target copilot,kiro        # subset
python3 sync/sync.py --agent code-writer          # one agent
python3 sync/sync.py --out ~/code/my-service      # write into a different project
```

## Agent source format

Each `agents/*.md` file is markdown with YAML frontmatter:

```markdown
---
name: code-writer
description: Writes new code following project conventions.
model: sonnet                                  # optional: sonnet | opus | haiku | inherit
tools: [Read, Edit, Write, Bash, Grep, Glob]   # optional
---

You are a focused coding sub-agent. ...
```

Fields:

- **name** — kebab-case, stable identifier; becomes the filename on every target.
- **description** — one line; platforms use this for routing / selection.
- **model** — optional. Adapters map this to each platform's model id.
- **tools** — optional allowlist. Omit to inherit the parent's full toolset.

The markdown body is the system prompt.

## Platform caveats

- **Kiro** has no true sub-agent concept. The adapter emits
  `.kiro/steering/<name>.md` with `inclusion: manual`, which makes
  each agent available as on-demand context ("act as code-writer").
  It is *not* an isolated agent with its own tool allowlist — `tools`
  in the source file are ignored. Configure Kiro tools via
  `.kiro/settings/mcp.json` instead.
- **Goose** does not auto-discover `.goose/recipes/`. Either `cd`
  into that directory before invoking Goose, export
  `GOOSE_RECIPE_PATH=$PWD/.goose/recipes`, or point Goose at a GitHub
  repo via `GOOSE_RECIPE_GITHUB_REPO`.
- **Copilot** model names use the `Name (vendor)` format, e.g.
  `Claude Sonnet 4.5 (copilot)`. The adapter maps `sonnet` / `opus` /
  `haiku` automatically; anything else is passed through verbatim.
  Bash/shell tools have no stable first-party identifier, so they're
  dropped — wire shell up via an MCP server if you need it.

## Tools via MCP

Keep tool configuration out of the agent files. All four platforms
speak MCP, so define servers once in `mcp.json` at the repo root and
let each platform load them:

- Claude Code: `.mcp.json` (symlink to `mcp.json`)
- Copilot: `.vscode/mcp.json`
- Goose: `~/.config/goose/mcp.json` or in a recipe
- Kiro: `.kiro/settings/mcp.json`

## Requirements

- bash 4+ (for `agentify.sh`)
- Python 3.9+ with `PyYAML` (for `sync/sync.py`)
- `git` (optional; `agentify.sh` will `git init` if the target has no repo)

## Future work

A grab-bag of features and improvements worth picking up:

### More target platforms

- **Cursor** — `.cursor/rules/*.mdc` (project rules) and any
  forthcoming sub-agent format.
- **Aider** — `.aider.conf.yml` plus per-agent `--message-file`
  presets.
- **OpenAI Codex CLI** / **Codex Web** — once a stable per-agent
  config format lands.
- **Continue.dev** — `.continue/` config, custom commands, and
  context providers.
- **Windsurf / Cascade** — workspace rules + per-agent prompts.
- **Zed AI assistant** — `.zed/` settings and slash commands.
- **JetBrains AI Assistant** — project-level prompt library.
- **Sourcegraph Cody** — `.sourcegraph/` custom commands.
- **Tabnine / Codeium / Supermaven** — where each exposes
  per-project system prompts.

### Better scaffolding

- **Re-run safely**: detect existing `AGENTS.md` / `README.md` / `agents/`
  and offer a three-way merge or interactive overwrite, instead of the
  current "exists, skipping" behaviour.
- **Project-type packs**: opinionated bundles per stack
  (`--preset python-fastapi`, `--preset node-next`, `--preset go-svc`)
  that seed extra agents (`migrations-author`, `api-contract-reviewer`,
  `ui-screenshot-tester`, etc.) and stack-specific skills.
- **Template inheritance**: let projects extend a base template with
  their own `templates/overrides/` so org-wide conventions can ride
  on top.
- **Skill library**: ship a curated set of starter skills
  (`run-tests`, `bisect-failure`, `add-migration`,
  `audit-dependencies`) instead of just `project-overview` /
  `run-tests`.

### Sync improvements

- **Round-trip / reverse-sync**: detect when a platform-native file
  has been hand-edited and offer to fold the changes back into
  `agents/<name>.md`.
- **Diff mode**: `sync.py --diff` showing what *would* change vs.
  what's currently on disk, per platform.
- **Watch mode**: `sync.py --watch` for live re-rendering during
  agent authoring.
- **Lint**: schema validation for frontmatter (required fields,
  unknown keys, model/tool name typos) with friendly error
  messages.
- **Per-platform overrides**: allow `agents/code-writer.copilot.md`
  to override a single field on one platform without forking the
  whole agent.
- **MCP server fan-out**: read `mcp.json` once and write each
  platform's expected location (`.mcp.json`,
  `.vscode/mcp.json`, `.kiro/settings/mcp.json`, ...) to keep
  toolsets in sync.

### Distribution & DX

- **Install script** / Homebrew tap / `pipx` package so `agentify`
  is available as a global command.
- **`agentify doctor`**: diagnose missing dependencies, broken
  symlinks, drifted platform files, and unreachable MCP servers.
- **`agentify update`**: pull the latest canonical agents/templates
  from upstream into a project that was scaffolded earlier.
- **CI template**: a GitHub Actions workflow that runs `sync.py
  --dry-run` on PRs to fail builds when platform files drift from
  canonical sources.
- **Tests**: golden-file tests for each adapter so platform output
  changes are reviewed deliberately.
